# Parking Permit App
A parking permit app that calculates the cost of a parking permit for a school/university. Takes into account permit type, vehicle type, carpool discounts, and number of months the user is paying for.

## Requirements
- Java JDK 25.0.2

## How to Run
Run `ParkingPermitApp.java` as the main class.

## Design
I followed the suggested structure of the assignment, as well as adding the `CampusFee` class. `ParkingPermitApp` functioned as my main. All printing was confined to `ParkingPermitApp` and `Receipt`, with `ParkingPermitApp` taking all the user input. Input is validated in a loop until the user gives a valid input, with all validation errors thrown from domain classes and caught in `ParkingPermitApp`.

Menus are dynamically generated by iterating over enum values, so adding a new permit or vehicle type automatically appears in the menu without any changes to `ParkingPermitApp`.

Pricing is handled through a strategy pattern. `ResidentPricingStrategy` and `CommuterPricingStrategy` implement `PricingStrategy` to determine the base monthly rate.

`RateModifier` is used as an interface to enforce the `apply` method on `CampusFee`, `CarpoolDiscount`, and `VehicleType`. `PricingPipeline` holds a list of a transaction's modifiers of type `RateModifier` and applies them in order.

`Validatable` is implemented on `PermitSelection` to validate user input before any calculations are performed, and on `PricingCalculator` and `PricingPipeline` to guard against invalid state in the business logic layer.

Exemplified polymorphism in the unit tests, as `testResidentBasePrice`, `testCommuterBasePrice`, and `testSUVModifier` all implement `RateModifier`.
